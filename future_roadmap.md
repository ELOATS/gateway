# Future Roadmap

本文档只保留真正还没有完成、但对长期维护有价值的下一阶段工作。已完成的主干收边界工作不再重复列为路线图。

## 已完成的基础能力

当前已经具备：

- 主链路分层与 application service 收口。
- Python / Rust 依赖 facade 化。
- 默认策略文件与集中路径配置。
- Proto 契约测试与生成物同步校验。
- 边界文档与 CI 架构守卫。

## 下一阶段重点

### 1. 跨语言集成测试

目前已有 Go 侧契约测试与 proto 同步检查，但仍然值得增加更真实的跨语言集成测试，例如：

- Go 启动后对 Python `CheckInput`、`CheckOutput`、`GetCache` 的端到端行为测试。
- Rust Nitro 输入检查和 token 统计的集成场景测试。

### 2. 部署成熟度提升

- 为本地开发、测试和 K8s 部署统一镜像与版本策略。
- 把依赖健康、降级语义与 readiness 的映射进一步标准化。
- 完善多环境配置模板与 Secret 管理说明。

### 3. 文档持续治理

- 当边界变更时，要求 PR 同步更新对应文档。
- 让开发文档和代码阅读指南保持最小延迟。
- 将运维、架构和契约文档纳入更严格的 review checklist。

### 4. 更细的运行态治理

- 扩展运行时依赖健康面板。
- 增加更清晰的 degraded 原因输出。
- 持续优化审计、事件和错误映射的一致性。

## 不再列为路线图的事项

以下事项已经完成，不应再当作待办：

- `ChatHandler` 下沉编排职责。
- 缺失 `policies.yaml` 导致测试失败的问题。
- 基础 proto descriptor 契约测试。
- CI 对关键边界文档和 proto 同步的自动检查。
