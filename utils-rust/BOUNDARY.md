# Rust Boundary

本文档定义 `utils-rust` 在当前 Gateway 中的稳定边界。

## 1. 定位

Rust Nitro / Wasm 能力当前属于基础能力层，不属于业务编排层。它存在的价值是提供高性能、确定性较强的底层能力。

## 2. 当前稳定职责

### 输入安全检查

Rust 可以承担高性能输入检查、敏感规则匹配或预处理。

要求：

- 返回结果语义稳定。
- 不要在 Rust 内隐式决定整个请求链路的业务命运。
- 若行为变化影响 Go 侧判断，必须同步更新 Go facade 行为测试。

### Token 统计

Rust 可以提供高性能 token 统计能力。

要求：

- 统计逻辑与模型映射语义应可解释。
- 对错误与不可用状态要有明确返回方式。

### 规则加载

Rust 可以维护敏感规则或 Wasm 所需的高性能运行资源。

要求：

- 规则更新方式应稳定。
- 不能让 Go 端依赖未声明的内部格式细节。

## 3. 不属于 Rust 的职责

- 请求主链路编排。
- HTTP 或 SSE 协议处理。
- Provider 路由与业务级策略决定。
- 多依赖综合降级决策。

## 4. 变更规则

如果需要扩展或修改 Rust 对外能力：

- 先判断它是否真的属于基础能力层。
- 如影响跨语言接口或稳定语义，同步更新 proto、Go facade、测试和文档。
- 不要通过临时隐式行为把 Rust 变成新的业务协调中心。

## 5. 与 Go 的协作方式

Go 侧推荐只通过 dependency facade 使用 Rust 能力。这样才能让超时、错误映射和降级行为保持统一。
